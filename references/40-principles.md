# 40 발명원리 — 전체 + 사업 적용

| # | 원리 | 핵심 | 사업 적용 예시 |
|---|------|------|---------------|
| 1 | 분할 | 대상을 독립 부분으로 분리 | 서비스 번들 → 개별 과금 |
| 2 | 추출 | 방해 속성만 분리·제거 | 핵심 기능만 MVP로 출시 |
| 3 | 국소 품질 | 균일→부분별 최적화 | 고객 세그먼트별 차등 UX |
| 4 | 비대칭 | 대칭→비대칭으로 전환 | 프리미엄/무료 비대칭 가격 |
| 5 | 통합 | 동종·인접 기능 결합 | 올인원 SaaS 플랫폼 |
| 6 | 범용성 | 하나가 여러 기능 수행 | 스마트폰(전화+카메라+지갑) |
| 7 | 포개기 | 하나 안에 다른 것을 넣음 | 플랫폼 위의 마켓플레이스 |
| 8 | 균형추 | 해로운 힘을 상쇄하는 힘 추가 | 환불 보장으로 구매 불안 상쇄 |
| 9 | 사전 반대 조치 | 예상 해로운 효과에 미리 대응 | 론칭 전 위기 대응 매뉴얼 |
| 10 | 사전 조치 | 필요한 변화를 미리 수행 | 사전 온보딩 프로세스 |
| 11 | 사전 보호 | 신뢰성 부족 보완 장치 | 자동 백업·롤백 시스템 |
| 12 | 등전위 | 작업 조건 변경으로 힘 불필요화 | 셀프서비스로 지원팀 부하 제거 |

| 14 | 곡면화 | 직선→곡선, 평면→구면 | 선형 프로세스→반복 피드백 루프 |
| 15 | 역동화 | 고정→가변·적응적으로 | 고정 가격→다이나믹 프라이싱 |
| 16 | 부족·과잉 조치 | 약간 더/덜 수행 후 조정 | MVP 과잉 출시→데이터 기반 축소 |
| 17 | 차원 전환 | 1D→2D→3D 이동 | 텍스트→인포그래픽→영상 콘텐츠 |
| 18 | 기계적 진동 | 진동·주기적 변화 활용 | A/B 테스트 주기적 반복 |
| 19 | 주기적 조치 | 연속→주기적으로 전환 | 상시 모니터링→주기적 감사 |
| 20 | 유용한 작용의 연속 | 비가동 시간 제거 | 대기 시간에 교육 콘텐츠 제공 |
| 21 | 고속 처리 | 빠르게 통과 | 빠른 실패(Fail Fast) 전략 |
| 22 | 전화위복 | 해로운 요소를 유익하게 | 클레임→고객 충성도 전환 기회 |
| 23 | 피드백 | 결과를 입력에 반영 | 사용자 행동 데이터→제품 개선 루프 |
| 24 | 매개체 | 중간 객체로 연결 | 플랫폼이 공급자-수요자 매개 |
| 25 | 셀프 서비스 | 대상이 스스로 기능 수행 | 고객이 직접 문제 해결(FAQ·챗봇) |

| 31 | 다공성 | 다공질·공극 활용 | 조직 내 비공식 커뮤니케이션 채널 |
| 32 | 색상 변경 | 가시성·투명도 변경 | 대시보드 시각화·상태 색상 코드 |
| 33 | 동질성 | 상호작용 대상과 동일 재료 | 고객이 고객을 지원(커뮤니티) |
| 34 | 배제·재생 | 사용 후 복원·재활용 | 리퍼비시·리셀 비즈니스 모델 |
